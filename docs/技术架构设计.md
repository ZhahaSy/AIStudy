# AI学习平台 - 技术架构设计

## 1. 系统架构概述

采用前后端分离的架构设计，使用统一的API服务支持多端客户端。

```
┌─────────────────────────────────────────────────────────────────┐
│                         客户端层                                 │
├─────────────────┬─────────────────┬─────────────────────────────┤
│   Web端 (PC)    │   移动端 (H5)   │      桌面客户端            │
│  React + TS    │   React移动适配  │   Electron + React        │
└────────┬────────┴────────┬────────┴────────────┬────────────────┘
         │                 │                     │
         └─────────────────┴─────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────────┐
│                        API网关层                                 │
│                    Nginx / API Gateway                          │
│              (负载均衡、限流、认证、路由)                        │
└────────┬────────────────────────────────────────────────────────┘
         │
         ┌─────────────────────────────────────────────────────────┐
         │                      服务层                              │
┌────────┴────────┐ ┌─────────────┐ ┌─────────────┐ ┌───────────┐
│   用户服务      │ │  学习资料服务 │ │ 学习计划服务 │ │ 题目服务  │
│   (User)       │ │  (Material)  │ │  (Learning) │ │ (Quiz)    │
└────────┬────────┘ └──────┬──────┘ └──────┬──────┘ └─────┬─────┘
         │                 │               │              │
┌────────┴─────────────────┴───────────────┴──────────────┴─────┐
│                        AI服务层                                 │
│              AI解析 │ 学习辅导 │ 题目生成 │ 答疑              │
└────────────────────────────┬───────────────────────────────────┘
                             │
┌────────────────────────────┴───────────────────────────────────┐
│                        数据层                                   │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────────────┐  │
│  │MySQL    │  │Redis    │  │对象存储  │  │  文件系统       │  │
│  │(主数据) │  │(缓存)   │  │(OSS/S3) │  │  (资料PDF等)    │  │
│  └─────────┘  └─────────┘  └─────────┘  └─────────────────┘  │
└───────────────────────────────────────────────────────────────┘
```

## 2. 技术栈选型

## 3. 核心模块设计

### 3.1 用户模块 (User Service)

**功能**：
- 用户注册/登录
- 个人信息管理
- 学习记录统计

**API设计**：
```
POST   /api/auth/register     - 注册
POST   /api/auth/login        - 登录
GET    /api/user/profile      - 获取用户信息
PUT    /api/user/profile      - 更新用户信息
GET    /api/user/stats        - 学习统计数据
```

### 3.2 学习资料模块 (Material Service)

**功能**：
- 资料上传与解析
- 资料库管理
- AI解析提取知识点

**API设计**：
```
POST   /api/materials/upload      - 上传资料
GET    /api/materials             - 获取资料列表
GET    /api/materials/:id         - 获取资料详情
DELETE /api/materials/:id        - 删除资料
POST   /api/materials/:id/analyze - AI解析资料
```

### 3.3 学习计划模块 (Learning Service)

**功能**：
- AI生成学习计划
- 学习进度管理
- 章节解锁控制

**API设计**：
```
POST   /api/learning/plans           - 创建学习计划
GET    /api/learning/plans/:id       - 获取学习计划
PUT    /api/learning/plans/:id       - 更新学习计划
GET    /api/learning/plans/:id/progress - 获取学习进度
POST   /api/learning/chapters/:id/complete - 完成章节
```

### 3.4 学习流程模块 (Study Service)

**功能**：
- 初次学习（知识点讲解）
- 答疑互动
- 成果检验

**API设计**：
```
GET    /api/study/chapter/:id/content     - 获取章节知识点
POST   /api/study/chat                    - 答疑对话
POST   /api/study/quiz/generate           - 生成测验题
POST   /api/study/quiz/submit             - 提交答案
GET    /api/study/quiz/:id/result         - 获取测验结果
```

## 4. 数据库设计

### 4.1 核心表结构 (MySQL)

```sql
-- 用户表
CREATE TABLE users (
    id              VARCHAR(36) PRIMARY KEY,
    email           VARCHAR(255) UNIQUE,
    phone           VARCHAR(20),
    password_hash   VARCHAR(255),
    nickname        VARCHAR(100),
    avatar          VARCHAR(500),
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 学习资料表
CREATE TABLE materials (
    id              VARCHAR(36) PRIMARY KEY,
    user_id         VARCHAR(36) NOT NULL,
    title           VARCHAR(255),
    file_url        VARCHAR(500),
    file_type       VARCHAR(20),
    status          VARCHAR(20) DEFAULT 'pending',
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 知识点表
CREATE TABLE knowledge_points (
    id              VARCHAR(36) PRIMARY KEY,
    material_id     VARCHAR(36) NOT NULL,
    chapter         VARCHAR(100),
    chapter_index   INT,
    title           VARCHAR(255),
    content         TEXT,
    summary         TEXT,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (material_id) REFERENCES materials(id)
);

-- 学习计划表
CREATE TABLE learning_plans (
    id              VARCHAR(36) PRIMARY KEY,
    user_id         VARCHAR(36) NOT NULL,
    material_id     VARCHAR(36) NOT NULL,
    status          VARCHAR(20) DEFAULT 'in_progress',
    target_level    VARCHAR(50) DEFAULT 'basic',
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (material_id) REFERENCES materials(id)
);

-- 学习进度表
CREATE TABLE learning_progress (
    id              VARCHAR(36) PRIMARY KEY,
    plan_id         VARCHAR(36) NOT NULL,
    knowledge_point_id VARCHAR(36) NOT NULL,
    status          VARCHAR(20) DEFAULT 'locked',
    quiz_score      INT,
    completed_at    TIMESTAMP NULL,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (plan_id) REFERENCES learning_plans(id),
    FOREIGN KEY (knowledge_point_id) REFERENCES knowledge_points(id)
);

-- 答疑记录表
CREATE TABLE qa_records (
    id              VARCHAR(36) PRIMARY KEY,
    user_id         VARCHAR(36) NOT NULL,
    plan_id         VARCHAR(36) NOT NULL,
    question        TEXT,
    answer          TEXT,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (plan_id) REFERENCES learning_plans(id)
);

-- 测验题目表
CREATE TABLE quiz_questions (
    id              VARCHAR(36) PRIMARY KEY,
    knowledge_point_id VARCHAR(36) NOT NULL,
    question_type   VARCHAR(20),
    question        TEXT,
    options         JSON,
    correct_answer  TEXT,
    explanation     TEXT,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (knowledge_point_id) REFERENCES knowledge_points(id)
);

-- 测验记录表
CREATE TABLE quiz_records (
    id              VARCHAR(36) PRIMARY KEY,
    user_id         VARCHAR(36) NOT NULL,
    plan_id         VARCHAR(36) NOT NULL,
    knowledge_point_id VARCHAR(36) NOT NULL,
    score           INT,
    answers         JSON,
    submitted_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (plan_id) REFERENCES learning_plans(id),
    FOREIGN KEY (knowledge_point_id) REFERENCES knowledge_points(id)
);
```

## 5. AI能力设计

### 5.1 资料解析

**输入**：PDF/Word/TXT文档
**处理流程**：
1. 文档转文本
2. AI提取章节结构
3. AI生成每个章节的精简知识点
4. 存储到knowledge_points表

### 5.2 学习计划生成

**输入**：资料内容、用户目标（学到什么程度）
**输出**：结构化的学习计划（章节顺序、预计时间）

### 5.3 知识点讲解

**输入**：章节ID
**输出**：精炼的知识点内容（适合"知道就好"的程度）

### 5.4 答疑

**输入**：用户问题 + 上下文
**输出**：AI即时回答

### 5.5 题目生成

**输入**：知识点ID
**输出**：3-5道测验题（选择题、判断题）
